Handtris

2024.06.17 ~ 2024.07.20

핸드 모션을 활용한 실시간 1:1 대결 테트리스 게임

Media Pipe의 핸드 모션 인식을 활용한 실시간 1:1 대결 테트리스 게임입니다.
기존 테트리스 게임과는 다르게 조작 컨트롤러나 키보드를 이용하지 않고 손동작을 이용하여 테트리스 블록을 조작하는 방식으로
사용자가 단순히 게임을 플레이하는 것뿐 아니라, 몸을 사용하여 더욱 적극적으로 게임에 참여할 수 있는 색다른 경험을 제공하는 것을
목표로 기획하게 되었습니다.
팀 구성은 백엔드 4명, 프론트엔드 2명으로 총 6명으로 구성된 팀 프로젝트이고 저는 백엔드를 담당하였습니다.
WebSocket의 동작 방식, pub/sub 구조와 STOMP 프로토콜에 대해 깊이 있게 공부할 수 있었고
Redis를 사용하여 API 응답속도를 개선하는 경험도 하였습니다.
또한 API 명세서 작성, CORS 설정 등 프론트엔드와 협업하는 과정에서 많은 것을 배울 수 있었습니다.

❮ ❯


💡담당한 기능


  • WebSocket `STOMP` 프로토콜을 사용하여 실시간성 보장
    • 테트리스 블록의 좌표값, 컬러 등의 정보를 Text 형식으로 상대방 유저에게 전달
  • Redis 캐시를 이용하여 API 호출 시간 단축
    • 게임방과 유저 정보를 매핑해 저장
    • 유저가 나가면 Redis에서 정보 삭제
  • ChannelInterceptor를 이용해 요청 전처리
    • 메시지를 가로채 요청 확인 및 수정
    • 나의 블록 정보를 보낼 때, 나도 그 메시지를 받아서 화면에 나의 블록과 상대방의 블록이 겹쳐서 그려지는 현상 발생
      • Interceptor에서 상대방 정보를 Header에 담아 블록 정보 전달 API에 전달
      • 블록 정보 전달 API에서 Header에 담긴 상대방에게만 메시지를 보내도록 수정
    • 비정상적인 종료로 인한 연결 해제 요청일 경우 API를 호출하지 않아 Redis와 DB의 정합성이 깨지는 현상 발생
      • 소켓 연결이 해제되면 클라이언트에서 특정 메시지 전송
      • Interceptor에서 이를 확인하고 정상적으로 API를 호출할 수 있도록 유도
  • Junit과 Mockito를 이용한 유닛 단위 테스트 작성

깃허브 : https://github.com/otfeb/BE-HandTris
